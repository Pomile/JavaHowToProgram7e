System Requirements

NOTES:
Since this installer is bundled with an older version of the JDK; you must have the most recent (6.0) version of Sun's Java™ 2 SDK installed, prior to installing this software.
The Sun Java System Application Server Platform Edition 8.2 and a database are bundled and installed with the product. The Java 2 Platform, Standard Edition (J2SE Platform) 1.5.0_04 software is bundled and installed for all supported operating systems except Mac OS. The browser settings for all platforms should be set with JavaScript enabled, cookies on, and CSS enabled.
The Sun Java Studio Creator 2 IDE supports the JDK 1.4 source level for code written within a project.

This CD includes the installer for the Java Studio Creator and requires the following...

Platform

  • Login as the root user (or as a user with the correct permissions, a.k.a. superuser).
  • Intel Pentium 4 (or equivalent) at 1 Ghz processing speed.
  • 1 GB of memory.
  • 750 MB (an additional 350 MB required for installation) of free disk space.
  • Operating system: Red Hat Enterprise Linux 3
  • Supported browsers:
    – At least Firefox 1.0.4

Database Servers and JDBC Drivers

  • Supported with the Java DB JDBC driver: Java DB - Sun's distribution of the open source Apache Derby 100% Java technology database - is bundled and installed with the IDE, and provided for samples only.
  • Supported with the included Sun Java System JDBC drivers:
    – Oracle (8iR3, 9iR2, and 10gR1)
    – Microsoft SQL Server 2000 SP3
    – Sybase ASE 12.5
    – IBM DB2 V8.1
  • Supported with other drivers: MySQL database server, 4.1 and 5.0, with JDBC driver MySQL Connector/J version 3.1.11

Deployment Servers

  • Supported deployment servers are listed below.
    – Sun Java System Application Server 8.2 PE (bundled)
    – Sun Java System Application Server 7.1 SE/EE
    – Sun Java Web Server 6.1
    – Sun Portal Server 6.3.1, 7.0
    – Apache Tomcat 5.x
    – BEA WebLogic Server 8.1
    – BEA WebLogic Server 7
    – IBM WebSphere 6

Using a Prior Version

You cannot upgrade to Java Studio Creator 2 Update 1 from any prior version of the IDE.

If you are using Sun Java Studio Creator 2, back up your data files and visit the Sun Java Studio Creator site to review the most up to date migration instructions prior to installing Java Studio Creator 2 Update 1.

For FAQs pertaining to the migration of applications developed with earlier versions of the IDE, visit the Sun Java Studio Creator site for detailed information.

If you are using the 2004Q2 release or any Early Access Release versions of the product, back up your data files, install Java Studio Creator 2 Update 1, and import the files into the new installation, as described below.

To preserve your data sources, you need to export your data sources before installing the new release.

  1. In the Servers window, right click on a Data Source and select Export Data Sources.
  2. In the Export Data Sources dialog box, save your data source in a directory that is independent of the Java Studio Creator user directory. When you have installed the new version of the IDE, you can import your data sources.

Before You Begin

  • Verify that your system meets the system requirements.
  • Check the FAQs page on the Java Studio Creator web site for late-breaking information.

Important: Before you install Java Studio Creator 2 Update 1, be sure to close any previous installations of the IDE on the same system. User settings do not migrate properly otherwise.

Installation Instructions

To install Java Studio Creator follow the instructions below.

  1. Download the creator-2_1-linux-ml.bin file for the Java Studio Creator IDE from this CD into a writable folder on your system.
  2. Change the file you downloaded to an executable file by typing:
    chmod +x creator-2_1-linux-ml.bin
  3. Start the installer by typing:
    ./creator-2_1-linux-ml.bin
  4. Follow the instructions in the installation wizard to complete the installation.

Starting the IDE

Go to $HOME/sun/Creator2_1/bin/creator if you used the default installation directory. For a non-default installation directory, go to install-directory/bin/creator.

You can use startup switches when you start the Creator IDE from the command line. The following table lists the startup switch and its action.

 

Startup Switch Description
--h
--help		 Print descriptions of common startup parameters.
--cp:p additional_classpath Prefix the specified classpath to the IDE's classpath.
--cp:a additional_classpath Append the specified classpath to the IDE's classpath.
--fontsize size Set the font size, expressed in points, in the IDE's user interface. If this option is not used, the font size is 11 points.
--jdkhome jdk_home_dir Note: The IDE is tested and bundled with the supported version of the Java 2 SDK, and in most instances, you do not need to use this switch. For all supported platforms except Mac OS, the default JDK is the JDK bundled with the product, located under $installdir/java. For the MacOS, the default JDK is the latest JDK installed on the machine

Use the specified version of the Java 2 SDK instead of the default SDK.

Back up your user directory before you upgrade the SDK that the IDE uses. If you later need to revert to the previous JDK, switch to the backed up user directory to ensure that you do not lose any settings.

To switch the IDE's user directory, use the -userdir switch.
--Jjvm_option Pass the specified flag directly to the JVM.
--locale language[:country[:variant]]

Use the specified locale. The supported languages are
English (en), Spanish (es), Korea (ko), Japanese (ja), French (fr), and Simplified Chinese (zh_CN).
For example, from the %SystemDrive%:\Program Files\Sun\Creator2\bin directory, type:
creator.exe --locale es
--userdir userdir Explicitly specify the userdir, which is the location in which user settings are stored.
If this option is not used on a UNIX system, the location is $HOME/.Creator/2_1 by default.

On Microsoft Windows systems, the default is the one specified when you first launched the IDE. In most cases, it is %SystemDrive%:\Documents and Settings\username\.Creator\2_1.

You can determine the current user directory in the About dialog box. Choose Help > About and then select the Detail Tab. The tab lists the location of the User Dir and other product details.

Troubleshooting Installation

All Supported Platforms

The following table shows installation issues that might occur on all supported platforms. The table also provides a workaround for each issue.

 
All Supported Platforms: Installation Issues and Workarounds
Problem Workaround
You get an insufficient temporary disk space error message while installing the product. The installer requires at least 500 MB of additional temporary disk space. On Windows XP and Windows 2000, the default temporary location is %SystemDrive%:\Documents and Settings\username\Local Settings\Temp. On UNIX platforms, the temporary space location is /var/tmp or /tmp.
Before you run the installer, ensure that you have at least 500 MB of temporary disk space in the locations listed. If you do not have enough disk space, then invoke the installer using the -is:tempdir option as shown below:
install-filename -is:tempdir temporary-directory
where install-filename is the file you downloaded.
The install wizard exits without installing the product. No messages are displayed. The downloaded file is not complete. Download the file again and check that the size of the downloaded file is the same as the file size specified on the download page. Run the installation wizard again.
Your installation fails. Run the installer in debug mode as follows to see the details of the installation failure:
  1. Before running the installer, create a file in the download directory named as follows:
    Linux platform: creator-2_1-linux-ml.sp
    Mac OS platform: creator-2_1-mac-ml.sp
    Microsoft Windows platform: creator-2_1-windows-ml.sp
    Solaris OS (SPARC platform): creator-2_1-sol-sparc-ml.sp
    Solaris OS (x86 platform): creator-2_1-sol-x86-ml.sp
  2. In the new .sp file, turn on the debug mode by adding the following line:
    is.debug=1
  3. Run the installer from the command prompt or terminal window as:
    installerFilename -is:log log.txt
    Where installerFilename is the file name for your platform.

UNIX Platforms

The following table shows installation issues and the workarounds for all supported UNIX platforms: Solaris OS and Linux.  

UNIX Platforms Installation Issues and Workarounds
Problem Workaround
If you perform the installation as root and then run the IDE as a normal user, you encounter problems when the IDE tries to write/update files in the installation directory due to directory/file access restrictions. Install the IDE under the user account that you intend to use when you run the IDE.
You get this message: The installer is unable to run in a graphical mode. Try running the installer with the -console or -silent option. Check that the DISPLAY environment variable is set correctly. If you are installing on your local system, the DISPLAY environment variable should be set to your-machine-name:0.0.

Check that you have enabled access to the X server with the command xhost + your-machine-name.

Note: The -console command does not work with the Creator installer.
It takes a long time for the Install Shield to start, or you receive a message stating: The directory directoryname does not contain enough space to extract temporary files. Please specify a temporary directory using the -is:tempdir option. Use the -is:help option for more information.

Install using the -is:tempdir option. You can start the installer by using this command:

./creator-2_1-sol-sparc-ml.bin -is:tempdir /tmp

The installer fails and displays a message stating that there is not enough disk space to use for installation. However, the file system that you specified for installing the IDE has enough available disk space. (5003279) Determine if the file system you specified has a symbolic link to another file system with disk space that is not recognized. For example, in the Solaris platform, /export/home might have 2 GB of space and / might have 100 MB. The /opt directory is symbolically linked to /export/home.

If you specified /opt/Creator as the installation directory, the install wizard does not recognize the symbolic link to /export/home. The install wizard recognizes only the / directory, which is the target directory for /opt and has only 100MB.

To correct the problem, directly specify the local file system with the larger available disk space as the installation directory.

Uninstalling the Sun Java Studio Creator 2 Update 1 Software

  1. From the install-directory/_uninst directory, type: ./uninstall.bin
  2. Follow the instructions in the uninstall wizard to complete the uninstall.
  3. Delete the installation folder.

Post Uninstall Cleanup

If you previously installed Java Studio Creator and are having problems installing, try doing a post uninstall cleanup. The following section gives information about cleanup on the different platforms.

  1. Start the uninstaller:
    • Mac OS platform: install-folder/_uninst/uninstall.command
    • Solaris OS and Linux platforms: install-directory/_uninst/uninstall.bin
  2. Remove the install-directory by running the /bin/rm -rf install-folder command.
  3. Clean up the productregistry file. Note: For Solaris OS and Linux platforms, you must be root to complete this step.

    The Linux productregistry file is located in /var/tmp/installation-directory/productregistry.

All Platforms

  1. To find the user directory, choose Help > About Sun Java Studio Creator, and then open the Details tab. The user directory is listed near the bottom of the About Window. 
    For example, the Windows OS user directory is %SystemDrive%:\Documents and Settings\username\.Creator\2_1.
  2. Delete the .Creator\2_1 directory and its contents.
    Note    : The Creator layout preferences, project open history, and any remote Application Servers settings are deleted when you remove the directory.
    When you reinstall the product, the settings revert to the default settings.

General Issues and Bugs

This section describes some general issues and bugs. The complete list of the known issues and bugs provides suggested workarounds for the different areas of the Java Studio Creator IDE. The known issues and bugs are sorted by area.

Administering the Deployment Servers

The IDE provides some administration functions which include starting and stopping the server, and undeploying an application or resource. You can find these items on context menus for the various nodes under Deployment Server in the Server Navigator window.
Each deployment server has its own administration tools. To administer a deployment server, that deployment server must be running.

Information about port numbers is available in the install-directory/rave2.1/config/com-sun-rave-install.properties file. Use the number indicated by the adminPort= setting.

The default user name and password are: username: admin, password: adminadmin.

The administrative port for Application Server is not the same as the HTTP port (default 29080) that is used to access web pages.

To access administration tools for Application Server:

  • Go to http://localhost:25848/.
  • To open the administration console in the IDE, right-click the Deployment Server node and choose View Admin Console from the context menu.

By default, AppServer installation logs are found in %SystemDrive%:\Document and Settings\Username\Local Settings\Temp .

Application Server out-of-memory error

If you do a large number of deployments and you have a large number of projects deployed to the Application Server, you might encounter an out-of-memory error.

Workaround: First, reboot your machine. Next, start the Application Server. You might also need to undeploy any project not needed for your current development work.

Query Editor

The Query Editor supports a limited set of queries. For unsupported queries,  you must enter the sql yourself. The Query Editor's failure to parse such an sql query does not affect query execution.
